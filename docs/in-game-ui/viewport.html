Viewport

Before talking about the viewport json object, we need to address the size of the main viewport, the "virtual screen" size where everything will be rendered. The size of that main viewport is around 3840 x 2160 (varies a lot depending of the aspect ratio). This means that if you want a collection to occupy the top left corner you would have something like: 

{
  "Collection": {
    "Name": "BlankBottomPanel",
    "ViewPort": {
        "xorigin": 0,
        "yorigin": 0,
        "width": 1920,
        "height": 1080,
        "alignment": "TopLeft"
        },
    "Widgets": [
      {
        "Widget": {
          "Type": "Base",
          "Name": "Background",
          "ViewPort": {
            "xorigin": 0,
            "yorigin": 0,
            "width": 1920,
            "height": 3060,
            "alignment": "TopLeft"
          },
          "StateMaterials": {
              "StateNormal": {
              "Material": "TransparentColor",
              "Color": {
                "r": 0.8,
                "g": 0.1,
                "b": 0.1,
                "a":0.5
              }
              }
          },
          "ChildWidgets": []
          }
        }]
    }
}

Attributes

As you can see, the attributes are pretty self explanatory: xorigin and yorigin indicate the position of the collection or widget, width and height indicate the size.

Alignment

However, alignment might be misunderstood. It's effect varies if its a widget or a collection.

  1. If it is a collection it will "align" the viewport so that it scales taking into account where the collection is anchored to (Centre, Right, etc).
  2. On both a collection and widget it will select one of 9 invisible points in the parent's rectangle from which the current collection/widget will be offseted.

The reason why the widget's alignment doesn't do the first function is easy to understand. Widgets and collections come with a locked aspect ratio set by the defined width and height, so scaling is straightfoward, therefore no necesary alignment needed.

In the previous example an offset of 0 would work, because alignment would be TopLeft, but let's see how we would achieve similar results with different alignments(not the same due to alignment function in collections):

Viewport in widgets

Lets look now the difference with widgets (no alignment due to same aspect ratio). You will notice you can achieve the same result even if you use different aspect ratios and alignments. See how can I use TopLeft for everything.

Notice how...

  1. to set something on center you can offset with Centre by Parent's Size/2, or with TopLeft Parent's Size /2 - Child's Size/2
  2. to set something on right/bottom you can offset with Right or Bottom by Parent's Size, or with TopLeft Parent's Size - Child's Size

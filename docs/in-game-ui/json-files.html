Json files related to in-game UI

The in-game UI is modified mainly by changing the widgetui folder, which contains a bunch of jsons and the inner folder Textures, which contains the image files used for the sprites.

widgetui files

  1. widgetui\commandpanel.json: By default BottomLeft. It modifies the section containing the unit and building commands, as well as the visualizing information when selecting 1 or more buildings or units.It is also responsible for displaying replay stats and buttons.
  2. widgetui\resourcepanel.json: By default TopLeft. It displays resources, population and age indicator.
  3. widgetui\mapanel.json: By default BottomRight. It displays the map and it's related buttons.
  4. widgetui\Menupanel.json: By default TopRight. It displays the buttons that open menus (diplomacy, settings, techtree, etc).
  5. widgetui\scorepanel.json: By default BottomRight. Mostly known for displaying the scores,althoughIt is used for some other stuff (displaying eco units, military units and other).
  6. widgetui\blanktoppanel.json: It simply displays the top horizontal png.
  7. widgetui\blankbottompanel.json: It simply displays the bottom horizontal png.
  8. widgetui\notificationpanel.json: It displays the messages from other players as well as other additional information (resigns, advancements in age).
  9. widgetui\wordltimerpanel.json: It displays the in-game time.
  10. widgetui\spectatorprogresspanelmanager.json: It displays the replay technology progress per player.
  11. GameMsgPanel.json: This file controls notifications like pop limit reached, not enough resources, redemption or atonement required.
  12. TechTreePreviewPanel This is the part of the UI that appears when you hover the civ icon, displaying the civ's tech tree.

Non modifiable UI

  1. Tooltip:

    When you hover a button, like create a villager button, a tooltip appears. That tooltip doesn't have a corresponding json file, so it can't be modded. The user can choose between "Off", "Floating" and "Fixed", as well as the scale.

It would be nice to be able to move the fixed tooltip position so that is not in the way of other UI elements.

Widgets

UI's are filled with widgets, so it is important to understand them. To learn how to position the widgets see the viewports guide.

Functionality

One would think that the Functionality of a widget depends on the widget's "type" attribute, but that is not entirely true. You can create a button if you use the type "button", but it's utility is probably tied to one of the following:

  1. The json file: It needs to be in a specific file. e.g. Copy map buttons into blanktoppanel.json. You will notice it just wont work.
  2. The hierarchy: The widget needs to have a full path of parent widgets and collections to work.
    e.g. Puting WoodStorage widget one widget above, or inside an added child will make it not display the corresponding value.
  3. Deadzone: The widget will not work, will break the game, or will fully interactible if it doesn't have a deadzone behind.
    e.g. Maps can be in any file in any place, but they work in a weird way without a proper deadzone.
  4. It doesnt depend on the above: Honestly, I still haven't found a widget that can be used anywhere no matter what (other than displaying fixed text or images). e.g. Show fixed text, fixed backgrounds.
